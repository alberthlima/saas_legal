En lo que concierne a la transición a producción, se optó por utilizar `gunicorn` y emplear múltiples workers a objeto de soportar tráfico alto. El principal impedimento para esto fue una variable que utiliza el bot de telegram para realizar las consultas, más específicamente `/area` / `user_areas` que define el manual que se utilizará y realizar la investigación correspondiente. Si define el área a utilizar y a continuación hace su consulta, es posible que el valor de esta variable se pierda. Para evitar esto será necesario emplear un motor de bases de datos extra, se recomienda Redis. Sin embargo, a objeto de priorizar la eficiencia y ahorrar consumo de procesamiento en el servidor se optará por brindar acceso a usuarios específicos en la empresa, al mismo tiempo que se empleará este enfoque para determinar de forma automática el manual a consultar.
